路径配置说明 ​

✅ 已完成配置 ​

你的网站将部署在：http://blog.zer0fire.me/demo/ai-novel/

关键配置 ​

1. VitePress Base 路径 ​

在 .vitepress/config.mts 中已设置：

typescript
export default defineConfig({
  base: '/ai-novel/',  // 重要：子路径配置
  // ...
})

2. GitHub 仓库链接 ​

所有文件中的 GitHub 链接已更新为：

  • https://github.com/zer0fire/demo/ai-novel

3. 在线访问地址 ​

  • README.md: http://blog.zer0fire.me/demo/ai-novel/
  • DEPLOY.md: http://blog.zer0fire.me/demo/ai-novel/

部署说明 ​

如果使用 GitHub Pages ​

如果你的域名 blog.zer0fire.me 指向 GitHub Pages，需要：

  1. 确保 CNAME 配置正确

    • 在仓库根目录创建 public/CNAME 文件（如果需要）
    • 内容为：blog.zer0fire.me

  2. GitHub Pages 设置

    • 进入仓库 Settings > Pages
    • Source 选择 GitHub Actions
    • Custom domain 填写：blog.zer0fire.me

如果使用自定义服务器 ​

如果你使用 Nginx 或其他 Web 服务器：

Nginx 配置示例 ​

nginx
server {
    listen 80;
    server_name blog.zer0fire.me;
    
    location /ai-novel/ {
        alias /path/to/.vitepress/dist/;
        try_files $uri $uri/ /ai-novel/index.html;
        index index.html;
    }
}

Apache 配置示例 ​

apache
<VirtualHost *:80>
    ServerName blog.zer0fire.me
    
    Alias /ai-novel /path/to/.vitepress/dist
    <Directory /path/to/.vitepress/dist>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
        
        RewriteEngine On
        RewriteBase /ai-novel/
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /ai-novel/index.html [L]
    </Directory>
</VirtualHost>

  • [x] 所有 GitHub 链接更新为 zer0fire/demo/ai-novel
  • [x] README.md 中的在线地址更新
  • [x] DEPLOY.md 中的部署地址更新
  • [x] 构建测试通过

注意事项 ​

  1. base 路径必须以 / 开头和结尾

    • ✅ 正确：base: '/ai-novel/'
    • ❌ 错误：base: 'ai-novel' 或 base: '/ai-novel'

  2. 所有内部链接会自动添加 base 前缀

    • 配置中的 link: '/大纲' 会自动变成 /ai-novel/大纲
    • 不需要手动在链接中添加 /ai-novel/

  3. 静态资源路径

    • VitePress 会自动处理所有静态资源的路径
    • 图片、CSS、JS 都会自动添加 base 前缀
