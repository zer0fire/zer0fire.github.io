部署指南 ​

本地开发 ​

1. 安装依赖 ​

首先需要安装 Node.js（建议 v18 或更高版本）。

bash
# 安装项目依赖
npm install

2. 本地预览 ​

bash
# 启动本地开发服务器，默认在 http://localhost:5173（端口可能不同）
npm run docs:dev

3. 构建静态文件 ​

bash
# 构建到 .vitepress/dist 目录
npm run docs:build

# 预览构建结果
npm run docs:preview

GitHub Pages 部署 ​

首次设置 ​

  1. 推送代码到 GitHub

    bash
    git add .
git commit -m "Migrate to VitePress"
git push origin main

  2. 启用 GitHub Pages

    • 进入你的 GitHub 仓库
    • 点击 Settings > Pages
    • 在 Source 下选择 gh-pages 分支
    • 点击 Save

  3. 等待部署完成

    • 每次推送到 main 分支时，GitHub Actions 会自动构建并部署
    • 在 Actions 标签页可以查看部署进度
    • 部署完成后，访问 http://blog.zer0fire.me//demo/ai-novel/

自动部署流程 ​

GitHub Actions 配置文件位于 .github/workflows/deploy.yml，它会：

  1. 在每次推送到 main 分支时自动触发
  2. 安装依赖
  3. 使用 VitePress 构建静态网站
  4. 部署到 gh-pages 分支
  5. GitHub Pages 自动发布更新

手动触发部署 ​

如果需要手动触发部署：

  1. 进入 GitHub 仓库的 Actions 标签页
  2. 选择 Deploy VitePress site to Pages workflow
  3. 点击 Run workflow 按钮

自定义配置 ​

修改仓库地址 ​

编辑 .vitepress/config.mts 文件，更新 GitHub 链接：

typescript
export default defineConfig({
  themeConfig: {
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名/你的仓库名' }
    ]
  }
})

同时更新 README.md 和 index.md 中的链接。

自定义主题 ​

VitePress 支持自定义主题样式，可以创建 .vitepress/theme/index.ts 文件来扩展默认主题。

详见：https://vitepress.dev/guide/custom-theme

添加更多功能 ​

VitePress 内置了许多功能：

  • 本地搜索（已启用）
  • 深色模式（已启用）
  • 最后更新时间（已启用）
  • 代码高亮
  • Markdown 扩展

故障排查 ​

端口被占用 ​

如果 5173 端口被占用，VitePress 会自动使用下一个可用端口。

构建失败 ​

检查：

  • Node.js 版本是否 >= 18
  • 依赖是否正确安装
  • Markdown 文件语法是否正确

GitHub Actions 失败 ​

查看 Actions 日志，常见问题：

  • 分支名称不匹配（main vs master）
  • 权限问题（确保 Actions 有写入权限）
  • base 路径配置问题

更新内容 ​

每次更新小说内容后：

bash
git add .
git commit -m "更新第X章"
git push origin main

GitHub Actions 会自动重新构建和部署。

VitePress 优势 ​

相比 GitBook，VitePress 提供：

  • ⚡️ 更快的构建速度和热更新
  • 🎨 更现代的界面设计
  • 🔍 更好的搜索体验
  • 📱 更好的移动端适配
  • 🛠️ 更活跃的维护和更新
  • 💪 完全兼容 Node.js v22+
